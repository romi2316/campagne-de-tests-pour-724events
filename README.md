# 🧪 Campagne de tests – 724events 

## 🗂️ Contexte

Ce projet consiste à mener une campagne de tests sur la première version livrée du site **724events**, une agence événementielle.

Le projet inclut :
- L’analyse et la compréhension du site via une image Docker.
- L'exécution d’un **cahier de recette** structuré.
- Des **tests exploratoires** pour valider les parcours utilisateurs.
- Le suivi des **anomalies sur Jira**.
- La rédaction d’un **bilan de campagne**.

## ⚙️ Outils utilisés

- 🐳 Docker (installation et lancement du site)
- ✅ Notion (stratégie de test, cahier de recette)
- 🐞 Jira (suivi des bugs)
- 📄 PDF / Excel pour la documentation et le bilan

## 📋 Livrables fournis

- ✅ Cahier de recette rempli (Excel)
- 🐞 Board Jira exporté au format CSV
- 📊 Bilan de fin de campagne de test (PDF)

## 🔎 Objectifs de la campagne

- Vérifier la conformité du site avec les spécifications.
- Identifier les anomalies via tests fonctionnels et exploratoires.
- Documenter les résultats et assurer la traçabilité des anomalies.

## 📊 Résultats

- **Nombre total de cas de test** : 16  
- **Taux de réussite des tests** : ✅ **75%**  
- **Taux d’échec** : ❌ **25%**
